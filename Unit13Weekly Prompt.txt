Question I am answering: In Chapter 14, you will note that in the code example for the Button class, the author has included triple-quoted strings for the class and functions. In fact, in most of the code provided in the book, triple-quoted strings have been provided for modules, methods, functions, and classes. Discuss how these docstrings help programmers with their workflow.

I think docstrings help programmers a lot because they make code easier to come back to and understand later. In a project like Alien Invasion, once the game starts getting bigger with different files, classes, and methods, it gets way easier to forget what each part is supposed to do. A docstring gives a quick explanation right at the top, so I do not have to read every line just to remember the purpose of something. That helps a lot when I am debugging too, because sometimes I already forgot what I was doing like ten minutes ago when trying to learn something new for the next part of the program. 

Docstrings also help keep code organized. A module docstring can explain what the whole file is for, while a class or function docstring explains one specific part. That makes it easier for me or somebody else to read the code and follow the logic. I think they also push me to write better code, because if I cannot explain what a function does in one or two sentences, that is usually a sign I made it more complicated than it needed to be.

They are also useful because Python can display docstrings with tools like help(), so they are built into the workflow instead of just being random comments. Honestly, I need that kind of help because half my battle in programming is forgetting whether I needed " " or ' ' and trying not to randomly start thinking in Java types for no reason. Overall, docstrings improve readability, save time, and make maintaining code much less painful.
